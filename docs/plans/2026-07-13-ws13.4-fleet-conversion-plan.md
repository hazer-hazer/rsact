# WS13.4 Fleet Conversion Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Execute the 13.3-approved conditions (M1/M2 derive hardening, excluded-surfaces sweep) then convert the remaining widget fleet (~13 of ~15; Button+Flex landed in 13.2) to the `Build<W>` builder/widget split.

**Architecture:** Each widget splits into a transient `XBuilder` (build-only props, consumed at `Build::build`) and a lean retained `X` (runtime fields only), generated via `#[derive(Builder)]` in `rsact-macros`. The contract is `docs/plans/2026-07-09-ws13.1-builder-widget-split-spec.md`; Button (`rsact-ui/src/widget/button.rs`) and Flex (`flex.rs`) are the in-repo worked examples.

**Tech Stack:** Rust proc-macros (`syn`/`quote`, already deps of `rsact-macros`), rsact-ui widget layer, metrics-probe for re-baselining.

    use syn::parse_quote;

    fn derive(
        mut input: syn::DeriveInput,
    ) -> syn::Result<proc_macro2::TokenStream> {
        impl_builder(&mut input)

Run: `cd /Users/as/dev/rsact-ws13 && cargo test -p rsact-macros 2>&1 | tail -5`
Expected: `widget_plus_child_on_one_field_is_an_error` and `child_plus_children_on_one_field_is_an_error` FAIL (`unwrap_err` panics — no error is returned today).

- [ ] **Step 3: Implement the guard** — in the field loop of `impl_builder` (before the existing `if field.attrs...is_ident("widget")` check), restructure to:

```rust
    for field in fields {
        // Named-fields only (checked above), so `ident` is always present.
        let ident = field.ident.as_ref().unwrap();

        // M1: the three roles are mutually exclusive. A double-annotated field
        // would be silently double-processed (child-wired AND husk-moved). A
        // widget that genuinely needs to wire a child and keep its husk must
        // hand-write its `Build` impl (per-type coexistence is by design).
        let has_widget =
            field.attrs.iter().any(|a| a.path().is_ident("widget"));
        let child_attr =
            field.attrs.iter().find(|a| a.path().is_ident("child"));
        let children_attr =
            field.attrs.iter().find(|a| a.path().is_ident("children"));
        let roles = usize::from(has_widget)
            + usize::from(child_attr.is_some())
            + usize::from(children_attr.is_some());
        if roles > 1 {
            return Err(syn::Error::new_spanned(
                field,
                "a builder field can carry only one of #[widget], \
                 #[child(...)], #[children(...)]; child fields are build-only \
                 (the arena owns the child) — to wire a child AND keep its \
                 husk in the retained widget, hand-write the Build impl",
            ));
        }

        if has_widget {
            widget_ctor_fields.push(quote! { #ident: this.#ident });
        }

        if let Some(attr) = child_attr {
            /* existing #[child] body unchanged, using `attr` */
        }

        if let Some(attr) = children_attr {
            /* existing #[children] body unchanged, using `attr` */
        }
    }
```

(The `child`/`children` bodies are the existing ones — only the `find` calls hoist to the top of the loop.)

- [ ] **Step 4: Run tests, verify all pass**: `cargo test -p rsact-macros 2>&1 | tail -3` → 3 passed. Then the UI suite (the derive is exercised by Button/Flex): `cargo test -p rsact-ui --lib --features "std,embedded-graphics" -- --test-threads=1 2>&1 | tail -3` → 65/0.

- [ ] **Step 5: Commit**: `git add rsact-macros/src/lib.rs && git commit -m "WS13.4: M1 - derive guard: one role per builder field"`

- [ ] **Step 3: Implement** — three edits:

(a) register the attribute: `attributes(builds, widget, child, children, flags, layout)`.

(b) after the `fields` binding (before the field loop), resolve the layout body:

```rust
    // --- M2: #[layout(delegate = "field")] on the struct (optional). ---
    // Default: return the builder's own `layout` field. Delegate: return a
    // child field's layout (`Show`-style wrappers own no layout, spec §5.4).
    let layout_body = match input
        .attrs
        .iter()
        .find(|a| a.path().is_ident("layout"))
    {
        Some(attr) => {
            let nv: syn::MetaNameValue = attr.parse_args()?;
            if !nv.path.is_ident("delegate") {
                return Err(syn::Error::new_spanned(
                    &nv,
                    "expected #[layout(delegate = \"field\")]",
                ));
            }
            let syn::Expr::Lit(syn::ExprLit {
                lit: syn::Lit::Str(name),
                ..
            }) = &nv.value
            else {
                return Err(syn::Error::new_spanned(
                    &nv.value,
                    "expected a string literal naming a builder field",
                ));
            };
            let field_ident = syn::Ident::new(&name.value(), name.span());
            if !fields
                .iter()
                .any(|f| f.ident.as_ref() == Some(&field_ident))
            {
                return Err(syn::Error::new_spanned(
                    name,
                    format!("no field named `{}` on this builder", name.value()),
                ));
            }
            quote! { self.#field_ident.layout() }
        },
        None => {
            if !fields
                .iter()
                .any(|f| f.ident.as_ref().is_some_and(|i| i == "layout"))
            {
                return Err(syn::Error::new_spanned(
                    &input.ident,
                    "builder has no `layout` field; add one or use \
                     #[layout(delegate = \"field\")] to return a child's \
                     layout",
                ));
            }
            quote! { self.layout }
        },
    };
```

(c) the emitted method becomes:

```rust
            fn layout(&self) -> rsact_ui::layout::node::Layout {
                #layout_body
            }
```

Also extend the derive's doc comment with the new attribute line and the M7 note:

```rust
/// - `#[layout(delegate = "field")]` (struct, optional) — `Build::layout`
///   returns `self.field.layout()` instead of the default `self.layout`
///   (for `Show`-style wrappers that own no layout of their own).
///
/// Note: the RETAINED widget must not override `Widget::flags`/`debug_name` —
/// they are read once, pre-build, from `Build` (seeding `ElState`); a retained
/// override is silently dead. Declare them here (`#[flags]`/`#[builds]`).
```

### Task 4 (recipe): per-widget conversion — apply to each widget in the Task 5 checklist

This is the exact procedure each fleet task follows. Worked examples in-repo: `button.rs` (single child), `flex.rs` (reactive children + de-generic). Each widget is ONE task with its own commit.

- [ ] **Step 1: Read the widget file** and classify every field:
  - **build-only** → builder-only field: children `El`s (`#[child(single)]`/`#[children(reactive)]`), values consumed while constructing `Layout` in `new()` (e.g. an `axis` param — deviation §4.3: don't store what only `new` needs).
  - **runtime** (read by `render`/`on_event`/`layout`) → `#[widget]` field, kept on the retained struct: `layout: Layout`, style fns, state/value `Signal`s, callbacks.
  - If the retained struct would have NO field using `W` → add `ctx: PhantomData<W>` to BOTH structs, `#[widget]` on the builder's (deviation §4.2, `flex.rs` precedent).
- [ ] **Step 2: Write the failing shape test first** — mirror `flex_split_drops_children_and_phantom` (grep it in `rsact-ui/src`): assert the retained widget no longer carries the build-only fields (e.g. via `size_of` bound or field-absence compile-driver) and the page still builds/renders via `test_support::NullWtf`.
- [ ] **Step 3: Split**: rename `struct X` → `XBuilder` with `#[derive(Builder)] #[builds(X<W>)] #[flags(...)]` (move the flags the old `Widget::flags` returned; then DELETE the retained `flags`/`debug_name` overrides — they are dead, M7). Define lean retained `X`. Keep the public constructor shape: `impl X { pub fn new(...) -> XBuilder }` (Button precedent) so call sites keep reading `X::new(...)`.
- [ ] **Step 4: Move builder-trait impls** (`LayoutWidget`/`SizedWidget`/`BlockModelWidget`/`FontSettingWidget` — whichever the widget had) onto `XBuilder`; retained `Widget` impl keeps `layout`/`render`/`on_event` only (no `build` — the builder does it; 7.6 slice).
- [ ] **Step 5: 7.2 slice where applicable**: `Dir: Direction` type param → runtime `Axis` ctor arg (flex precedent); `V: RangeValue` → canonical numeric at the boundary (slider/knob/bar).
- [ ] **Step 6: Fix call sites**: `grep -rn "X::\|\.el()" rsact-ui/src rsact-ui/examples size-probe/src metrics-probe/src src/` — builders aren't `Widget`, so `.el()` → `.into_el()` (`View` method). Check size-probe + the Task-3 example baseline (fix only NEW breakage).
